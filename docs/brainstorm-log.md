# Brainstorm Log

Target Version: 3.3 (candidate)

## 2026-07-20 - 船只选择候选

### Candidate Direction

- 船型的价值应来自不同航路能力，而不只是速度、货舱依次变大。
- 推荐保留现有入门船，并形成四个明确角色：近海桨帆船、探索卡拉维尔、远洋货船、远洋护航船。
- 如果 3.3 只新增一种船，优先加入 **Fusta（轻型桨帆船）**：它能让无风、浅水港、短途接驳和港口排队产生新的路线选择。
- 将现有“轻快帆船”明确为 **拉丁帆卡拉维尔**，突出逆风、转向、地图探索和港口发现能力。
- **卡瑞克/Nau** 定位为大宗远洋货运：货舱最大，但转向慢、靠港装卸时间长。
- **盖伦** 不应只是货舱最大的上位船；候选定位是高速远洋护航和高风险航路船，货舱小于卡瑞克，后续可为同商会 NPC 航路提供护航收益。

### Candidate Roster

| 船型 | 主要玩法 | 优势 | 代价 |
| --- | --- | --- | --- |
| 巴尔沙 / 小型沿岸帆船 | 初始近海贸易 | 便宜、靠港快 | 货舱小、远洋能力差 |
| Fusta 轻型桨帆船 | 地中海短途、支线接驳 | 无风仍可低速前进、转向快、可进浅港 | 船员与补给消耗高、外海抗风浪差 |
| 拉丁帆卡拉维尔 | 探索与开图 | 逆风和转向强、发现范围大 | 货舱较小 |
| 卡瑞克 / Nau | 大宗跨洋贸易 | 货舱与续航最大 | 笨重、装卸慢、占用泊位久 |
| 盖伦 | 高速跨洋与护航 | 顺风航速、耐波性和航路安全 | 昂贵、货舱不应超过卡瑞克 |

### Open Questions

- 3.3 是只加入 Fusta，还是同时把现有船型重新命名并重新平衡？
- 无风最低航速、浅水港限制、船员工资、装卸时间和护航风险中，首版实现哪些机制？
- 是否等战斗或护航系统出现后再开放大型商用桨帆船（merchant galley）？

## 2026-07-20 - 商会伙伴与两级港口情报

### Decisions

- 阿尔瓦罗、索菲亚、伊德里斯从友好 NPC 升级为同商会伙伴。
- 伙伴航线使用独立样式显示在海图和航线列表中，伙伴船可继续跟随观看。
- 港口情报拆成“已有地图坐标”和“已亲自发现”两级状态。
- 伙伴航线经过的港口会获得地图坐标；玩家可按图航行至附近，但不能提前查看商品或加入自动贸易航线。
- 玩家亲自抵近后才解锁完整港口情报、贸易和自动航线权限。

### Rationale

- 同商会伙伴能把现有 NPC 自动航路转化为玩家可利用的情报网络，而不仅是背景交通。
- “知道坐标”不等于“完成实地发现”，能够表达地图、传闻与亲历之间的信息差。
- 伙伴航线为探索提供方向，但不替代玩家亲自航行。

### Out of Scope

- 3.2 暂不加入伙伴任务、好感度、分红、共同投资和护航指令。
- 暂不允许玩家编辑或删除伙伴航线。

### Success Criteria

- 伙伴航线在迷雾上方可见，并在航线列表中明确标注所属伙伴。
- 地图已标注但未发现的港口拥有不同图标与受限信息面板。
- 未亲自发现的港口仍无法加入玩家自动航线。
- 两级港口状态可存档，并兼容 3.1 存档。

## 2026-07-20 - 港口吞吐与 Gateway 类比

### Decisions

- 港口具有有限泊位：远洋前哨 1 个、普通港 2 个、区域枢纽 3 个。
- 玩家、玩家商船与 NPC 商船共享泊位，按抵达次序排队。
- 自动商船抵达后先在锚地等待，获得泊位后才完成装卸和交易。
- 玩家遇到满港时进入可取消的锚地等待状态。
- 海图和港口界面显示泊位占用与排队数量，使拥堵成为可观察、可规划的系统。

### Rationale

- 历史帆船港口受深水岸线、潮汐、驳船和人工装卸能力限制，拥堵是贸易网络扩张的真实约束。
- 该机制也是太空 Gateway 的合理类比：航线数量对应交通需求，泊位与装卸能力对应对接端口和货运吞吐能力。
- NPC 自动扩张航路后，热门节点会自然出现拥堵，促使玩家分流到次级港口，而非只选择最高利润路线。

### Out of Scope

- 3.1 暂不加入港口升级、付费插队、检疫、潮汐窗口、货损和补给消耗。
- 暂不模拟不同货物的独立装卸设备。

### Success Criteria

- 多船同时抵达同一港口时，超过容量的船不会同时交易。
- 队列按抵达顺序释放，存档后仍保持状态。
- 玩家能看到港口容量、占用和等待数量，并能取消自己的等待。

### Next Actions

- 未来版本可加入港口扩建、驳船服务、加急费与拥堵导致的货损。
